# 三方支付数据统计系统 - 需求文档

## 1. 项目概述

三方支付数据统计系统用于逐笔展示订单收款明细，识别三方支付、个人报销等异常场景，辅助风控人员进行资金风险排查。

## 2. 页面布局

采用左右分栏布局：
- **左侧（78%）**：产品功能界面，包含筛选区、规则提示、数据表格
- **右侧（22%）**：功能说明面板，对左侧各模块进行连线说明
- **SVG 连线层**：左右两侧对应模块通过贝塞尔曲线连线，鼠标悬停高亮联动

### 3.3 核心规则提示

页面展示核心规则摘要：
> 客户名称优先级：合同 > 发票 > 企业认证 > 月结 > 在线账号 |
> 三方支付：客户名称与付款名称不一致 |
> 个人报销：微信/支付宝/汇付天下/银联且付款人为空

## 4. 业务规则详解

### 4.1 三方支付判定

**定义**：客户名称与付款名称不一致即为三方支付。

**例外情况**：微信/支付宝/汇付天下/银联且付款人为空时，**优先判定为个人报销**，不归为三方支付。

### 4.2 个人报销判定

**定义**：订单支付方式为微信/支付宝/汇付天下/银联，且付款人为空时，判定为个人报销。

### 4.3 客户名称优先级

当同一客户存在多个名称来源时，按以下优先级取值：

```
合同名称 > 发票名称 > 企业认证名称 > 月结客户名称 > 在线账号名称
```

### 4.4 线上/线下判定

| 场景 | 判定结果 |
|-----|---------|
| 前台下过单 | 线上 |
| PCB/PCBA 名单客户 | 线下 |
| 兼有 | 线下 |

### 4.5 是否收款判定

依据**实际付款金额是否大于 0**来判断，而非应收生成。

- 实际付款金额 > 0：已收款
- 实际付款金额 = 0：未收款

### 4.6 付款名称取值规则

| 付款方式 | 付款名称取值逻辑 |
|---------|----------------|
| 公司转账 | 取真实打款人名称 |
| 汇付天下 | 取台账付款人名称 |
| 余额支付 | 按充值记录追溯付款人 |
| 信用支付 | 按还款方式追溯，未还款显示"未还款无付款人" |

### 4.7 多付款记录展示规则

每个订单如有多个付款名称及金额，需**分多行展示**，每行独立显示各自的付款名称、付款方式、支付流水号和付款金额。
